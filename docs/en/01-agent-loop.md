# 01 — Agent Loop Deep Dive: The 1729 Lines of query.ts

> Files: `src/query.ts` (1729 lines), `src/query/`, `src/services/tools/`

---

## 1. Overall Architecture

The core of Claude Code is an **infinite while loop**; each iteration follows this flow:

```
[Prepare messages] → [Context compaction] → [Call model API] → [Process streaming response] → [Execute tools] → [Loop continues / terminates]
```

The entire loop is an **AsyncGenerator** that yields various events to the caller (the REPL or the SDK), which consumes those events to update the UI or return results.

## 2. Entry Function

```typescript
// src/query.ts

export type QueryParams = {
  messages: Message[]
  systemPrompt: SystemPrompt
  userContext: { [k: string]: string }
  systemContext: { [k: string]: string }

  toolUseContext: ToolUseContext
  fallbackModel?: string
  querySource: QuerySource
  maxOutputTokensOverride?: number
  maxTurns?: number
  skipCacheWrite?: boolean
  taskBudget?: { total: number }  // API task budget (2026-03-13 beta)
  deps?: QueryDeps               // Dependency injection (for testing)
}

export async function* query(
  params: QueryParams,
): AsyncGenerator<
  | StreamEvent         // API streaming data (assistant messages, tool calls)
  | RequestStartEvent  // Request-start signal (triggers spinner)
  | Message            // Complete message (user/assistant/system)
  | TombstoneMessage   // Message deletion signal (on model switch)
  | ToolUseSummaryMessage,  // Tool call summary (generated by Haiku)
  Terminal             // Termination reason
>
```

## 3. Loop State

All mutable state in the loop is consolidated into a single `State` object, which is replaced wholesale at each `continue` point (functional-style update to avoid missing fields):

```typescript
type State = {
  messages: Message[]
  toolUseContext: ToolUseContext
  autoCompactTracking: AutoCompactTrackingState | undefined
  maxOutputTokensRecoveryCount: number      // number of max_output_tokens recovery attempts
  hasAttemptedReactiveCompact: boolean      // whether reactive compaction has been attempted
  maxOutputTokensOverride: number | undefined   // override for output token limit
  pendingToolUseSummary: Promise<ToolUseSummaryMessage | null> | undefined
  stopHookActive: boolean | undefined       // whether inside a stop hook loop
  turnCount: number                         // current turn count
  transition: Continue | undefined          // transition reason from previous iteration (for assertions)
}
```

**Transition reasons (`Continue.reason`):**

| Reason | Trigger condition |
|--------|-------------------|
| `max_output_tokens_escalate` | Output truncated; retry with limit raised from 8k to 64k |
| `max_output_tokens_recovery` | Output truncated; inject recovery prompt and continue (up to 3 times) |
| `collapse_drain_retry` | Context Collapse submitted a partial fold; retry API |
| `reactive_compact_retry` | Reactive Compact succeeded; retry API |
| `stop_hook_blocking` | Stop Hook injected a blocking error; must continue |
| `token_budget_continuation` | Token budget not exhausted; continue |
| `model_fallback` | Model switched (fallback to backup model during peak load) |

## 4. The 8 Phases of Each Iteration

### Phase 1: Initialization and Pre-checks

```typescript
yield { type: 'stream_request_start' }  // → REPL shows spinner

// Tool result budget: offload large results to disk (avoids context explosion)
messagesForQuery = await applyToolResultBudget(...)

// Skill discovery prefetch (parallel, non-blocking)
const pendingSkillPrefetch = skillPrefetch?.startSkillDiscoveryPrefetch(...)
```

### Phase 2: Snip (History Trimming)

```typescript
// Controlled by the HISTORY_SNIP feature flag
if (feature('HISTORY_SNIP')) {
  const snipResult = snipModule!.snipCompactIfNeeded(messagesForQuery)
  messagesForQuery = snipResult.messages
  snipTokensFreed = snipResult.tokensFreed  // passed to autocompact to adjust threshold
  if (snipResult.boundaryMessage) yield snipResult.boundaryMessage
}
```

### Phase 3: Microcompact (Inline Tool Result Compaction)

```typescript
// Compacts old tool results (retains the most recent N turns, clears older file reads, bash output, etc.)
const microcompactResult = await deps.microcompact(
  messagesForQuery,
  toolUseContext,
  querySource,
)
messagesForQuery = microcompactResult.messages
// Compactable tools: Read, Bash, Grep, Glob, WebSearch, WebFetch, Edit, Write
```

### Phase 4: Context Collapse (Progressive Folding)

```typescript
// CONTEXT_COLLAPSE feature flag
if (feature('CONTEXT_COLLAPSE') && contextCollapse) {
  const collapseResult = await contextCollapse.applyCollapsesIfNeeded(...)
  messagesForQuery = collapseResult.messages
  // Collapsing is a "view projection" — original messages are preserved in REPL history;
  // collapse summaries live in the collapse store and are replayed via projectView() each loop
}
```

### Phase 5: Autocompact (Automatic Conversation Compaction)

```typescript
// Check whether the threshold is exceeded; if so, summarize with Haiku
const { compactionResult, consecutiveFailures } = await deps.autocompact(
  messagesForQuery,
  toolUseContext,
  { ... },
  querySource,
  tracking,
  snipTokensFreed,
)

if (compactionResult) {
  // Compaction succeeded: yield compacted messages, continue loop
  const postCompactMessages = buildPostCompactMessages(compactionResult)
  for (const message of postCompactMessages) yield message
  messagesForQuery = postCompactMessages
}
```

**Autocompact threshold calculation:**

```typescript
// Effective context window = model max window − reserved output space (20,000 tokens)
effectiveContextWindow = contextWindow - min(maxOutputTokens, 20_000)

// Trigger threshold = effective window − 13,000 token buffer
autocompactThreshold = effectiveContextWindow - 13_000

// Warning threshold = trigger threshold − 20,000 (used for UI prompt)
warningThreshold = autocompactThreshold - 20_000

// Blocking limit (when autocompact is disabled) = 100% of effective window
blockingLimit = effectiveContextWindow
```

### Phase 6: Call Model API (Streaming)

```typescript
for await (const message of deps.callModel({
  messages: prependUserContext(messagesForQuery, userContext),
  systemPrompt,
  tools: toolUseContext.options.tools,
  signal: toolUseContext.abortController.signal,
  options: {
    model: currentModel,
    fallbackModel,
    querySource,
    // ... other params
  },
})) {
  // Handle each streaming event
  if (message.type === 'assistant') {
    assistantMessages.push(message)
    // Collect tool_use blocks
    const toolUseBlocks = message.message.content.filter(b => b.type === 'tool_use')
    if (toolUseBlocks.length > 0) needsFollowUp = true
  }

  // Withhold recoverable errors (don't yield yet; wait for recovery logic)
  if (!withheld) yield message
}
```

**Recoverable errors (withheld, not yielded immediately):**
- `prompt_too_long` (413) → attempt Context Collapse or Reactive Compact
- `max_output_tokens` (truncation) → attempt to escalate limit or inject recovery prompt
- Media size errors (image/PDF too large) → Reactive Compact removes media

**Model fallback (`FallbackTriggeredError`):**

```typescript
catch (innerError) {
  if (innerError instanceof FallbackTriggeredError && fallbackModel) {
    currentModel = fallbackModel
    attemptWithFallback = true  // retry the entire API call

    // Send tombstones for any already-yielded orphaned messages (so UI can remove them)
    for (const msg of assistantMessages) {
      yield { type: 'tombstone', message: msg }
    }
    // Clear and retry
    assistantMessages.length = 0
    continue
  }
}
```

### Phase 7: Execute Tools

```typescript
// Two execution paths:
// 1. StreamingToolExecutor (parallel streaming execution — execute while receiving stream)
// 2. runTools (serial — wait for stream to finish, then execute all at once)
const toolUpdates = streamingToolExecutor
  ? streamingToolExecutor.getRemainingResults()
  : runTools(toolUseBlocks, assistantMessages, canUseTool, toolUseContext)

for await (const update of toolUpdates) {
  if (update.message) yield update.message
  if (update.newContext) updatedToolUseContext = update.newContext
}

// After tool execution, generate tool summary in parallel (uses Haiku, non-blocking)
nextPendingToolUseSummary = generateToolUseSummary({ tools: toolInfoForSummary })
```

### Phase 8: Decide Whether to Continue the Loop

```typescript
if (!needsFollowUp) {
  // No tool calls → check stop hooks, token budget → terminate

  const stopHookResult = yield* handleStopHooks(...)
  if (stopHookResult.preventContinuation) return { reason: 'stop_hook_prevented' }
  if (stopHookResult.blockingErrors.length > 0) {
    // Stop hook injected errors → continue loop so model can handle them
    state = { ..., transition: { reason: 'stop_hook_blocking' } }
    continue
  }

  return { reason: 'completed' }  // normal completion
}

// There were tool calls → assemble next-turn messages, continue loop
if (maxTurns && nextTurnCount > maxTurns) {
  yield createAttachmentMessage({ type: 'max_turns_reached', ... })
  return { reason: 'max_turns' }
}

state = {
  messages: [...messagesForQuery, ...assistantMessages, ...toolResults, ...attachments],
  turnCount: turnCount + 1,
  // ...
}
// while(true) continues to next iteration
```

## 5. Termination Reasons (`Terminal.reason`)

```typescript
type Terminal = { reason:
  | 'completed'               // normal completion (no tool calls, stop hooks passed)
  | 'aborted_streaming'       // user interrupted during streaming (Ctrl+C)
  | 'aborted_tools'           // user interrupted during tool execution
  | 'max_turns'               // reached maximum turn limit
  | 'blocking_limit'          // context exceeded blocking threshold (when autocompact disabled)
  | 'prompt_too_long'         // 413 and unrecoverable
  | 'image_error'             // image/media error
  | 'model_error'             // internal model call error
  | 'stop_hook_prevented'     // Stop hook blocked continuation
  | 'hook_stopped'            // Hook sent STOP signal
}
```

## 6. Key Design Patterns

### 6.1 Dependency Injection (QueryDeps)

```typescript

  callModel: typeof callModel
  microcompact: typeof microcompact
  autocompact: typeof autoCompactIfNeeded
  uuid: () => string
}

// In tests, replace with mocks

  callModel: mockCallModel,
  microcompact: noopMicrocompact,
  autocompact: noopAutocompact,
  uuid: deterministicUuid,
}
```

### 6.2 Prompt Cache Protection

```typescript
// The "observable copy" of tool input is kept separate from the "API-sent content"
// backfillObservableInput only modifies the copy, not the original
// The original stays byte-for-byte identical → maximizes Prompt Cache hit rate

let yieldMessage: typeof message = message
if (message.type === 'assistant') {
  // Clone and populate legacy/derived fields, visible only to SDK/transcript
  // Original message (same bytes) is used when re-sending to API
}
```

### 6.3 Task Budget (new in 2026-03-13)

```typescript
// API-level task budget (measures context window consumption, not raw token count)
// Records "consumed context" at each compaction
taskBudgetRemaining = Math.max(
  0,
  (taskBudgetRemaining ?? params.taskBudget.total) - preCompactContext,
)
// Passed to the next API call so the server knows the remaining budget
```

## 7. Complete Flow Diagram

```
query()
  │
  └─ queryLoop()
       │
       while(true)
         │
         ├─ yield stream_request_start
         │
         ├─ 1. applyToolResultBudget (offload large tool results)
         │
         ├─ 2. snipCompactIfNeeded (HISTORY_SNIP)
         │
         ├─ 3. microcompact (inline tool result compaction)
         │
         ├─ 4. contextCollapse.applyCollapsesIfNeeded (progressive folding)
         │
         ├─ 5. autoCompactIfNeeded
         │      ├─ below threshold → skip
         │      └─ above threshold → summarize with Haiku → yield compacted messages
         │
         ├─ 6. callModel (streaming)
         │      ├─ yield assistant messages
         │      ├─ yield tool_use blocks (needsFollowUp = true)
         │      ├─ withhold recoverable errors (PTL, max_output_tokens, media)
         │      └─ model fallback → yield tombstones → switch model and retry
         │
         ├─ 7. executePostSamplingHooks (async, non-blocking)
         │
         ├─ 8. handle recoverable errors
         │      ├─ PTL → collapse drain → reactive compact → yield error
         │      ├─ max_output_tokens → escalate limit → inject recovery prompt
         │      └─ media error → reactive compact
         │
         ├─ 9. no needsFollowUp → stop hooks → token budget → terminate
         │
         ├─ 10. runTools / StreamingToolExecutor
         │       └─ yield tool results (including permission confirmation dialogs)
         │
         ├─ 11. fetch attachments (queued commands, memory attachments)
         │
         └─ 12. maxTurns check → assemble next-turn state → continue
```
